## 🌐 الرؤية الجوهرية (The Core Vision)
NEXUM ليس مجرد "شات بوت"؛ إنه **نظام تشغيل سيادي (Sovereign OS)** مصمم ليعمل كـ **Master Agent**. هو لا يكتفي بالحديث، بل يخطط، ينفذ، يبني، ويراقب. تم تصميمه ليكون المصنع الأول للوكلاء الذكيين (Sub-Bots) والمنصة المركزية لإدارة البنية التحتية البرمجية عبر التليجرام.

---

## 🧠 المحركات المركزية (Core Engines)

### 1. ⚙️ محرك التنفيذ الحي (Execution Engine)
يستقبل الأهداف باللغة الطبيعية، يحولها إلى خطط JSON معقدة، وينفذها خطوة بخطوة مع تقطير حي (Live Streaming) لكل عملية.
- **التخطيط**: تحويل الكلام إلى خطوات برمجية.
- **التحقق (Verifier)**: لا يبلغ عن النجاح إلا بعد التأكد من وجود النتيجة على القرص.
- **البث**: ترى تقدم العمل لحظة بلحظة.

### 2. 📂 وحدة التحكم الجراحي بالملفات (FS Control)
نظام متكامل مستوحى من أدوات سطح المكتب المتقدمة:
- **القراءة الذكية**: قراءة الأكواد بنظام السطور.
- **التعديل الجراحي**: إضافة، حذف، أو استبدال سطور برمجية دون لمس بقية الملف.
- **البحث العميق**: محرك Grep داخلي للبحث في المحتوى والأسماء والأحجام.
- **الملاحة الهرمية**: عرض بنية المشروع كاملة (Tree/LS).

### 3. 👁️ الرؤية المتقدمة (Vision & Multimodal)
مدعوم بـ **Gemini 2.0 Flash**، يستطيع النظام:
- تحليل الصور (لقطات الشاشة للأخطاء، التصاميم).
- تحليل المستندات والملفات البرمجية المضغوطة.
- الرد بسياق بصري وتقني متكامل.

### 4. 🤖 بروتوكول الوكلاء (Inter-Bot Protocol)
القدرة على إنشاء وتسجيل وإدارة جيش من الـ Sub-Bots المستقلة والتحكم بها مركزياً.

## 🛠️ أوامر التحكم السيادية

| الأمر | الوصف |
|-------|-------|
| `! [الأمر]` | تفعيل **محرك التنفيذ** المباشر (Direct Execution) |
| `/workspace` | عرض محتويات منطقة العمل النشطة |
| `/tree` | رؤية الهيكل الهرمي للمشروع كاملاً |
| `/ls [path]` | تصفح المجلدات والملفات |
| `/cat [file]` | قراءة محتوى ملف (مع دعم نطاق السطور) |
| `/find [query]` | البحث الشامل في الأسماء والمحتوى |
| `/bots` | إدارة سجل الوكلاء الذكيين (Sub-Bots) |
