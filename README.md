# 🔱 NEXUM CORE OS — نظام التشغيل السيادي
**الجيل القادم من أنظمة إدارة السيرفرات عبر التلجرام المدعومة بالذكاء الاصطناعي.**

---

## 📖 نظرة عامة (Overview)
**NEXUM CORE** ليس مجرد بوت تلجرام، بل هو بيئة تشغيل متكاملة (Sovereign OS) تتيح للمسؤولين التحكم الكامل في البنية التحتية للسيرفرات، وإدارة حاويات Docker، ونشر الأكواد (Deployment) باستخدام لغة طبيعية أو عبر واجهة رسومية متطورة، مع حماية أمنية مشددة تعتمد على الذكاء الاصطناعي.

---

## 🏗️ الهيكلية التقنية للمشروع (Project Architecture)

المشروع مبني بتصميم **Modular** (وحدوي) لسهولة التطوير والصيانة. إليك شرح لكل جزء:

### 1️⃣ النواة (Core Module - `/core`)
هذا هو "دماغ" النظام والمسؤول عن العمليات الحساسة:
- **`executor.py`**: المحرك المسؤول عن تنفيذ أوامر Linux في السيرفر. يتميز بنظام معالجة الأخطاء والمهلة الزمنية (Timeout).
- **`planner.py`**: محرك التخطيط الذكي. يأخذ "الهدف" من المستخدم (مثل: ثبّت nginx واجعله يعمل على بورت 80) ويحوله إلى خطة عمل تسلسلية بصيغة JSON.
- **`security.py`**: صمام الأمان. يحتوي على قوائم الأوامر المحظورة (Blacklist) والأوامر التي تتطلب تأكيداً يدوياً قبل التنفيذ.
- **`llm_factory.py`**: المسؤول عن إدارة الجلسات مع نماذج الذكاء الاصطناعي (Gemini 2.5 & GPT-4o) وضمان استمرارية السياق للمستخدم.
- **`keyboards.py`**: نظام الملاحة التفاعلي. يحتوي على جميع تعريفات الأزرار والقوائم التي تظهر في التلجرام.
- **`formatters.py`**: المسؤول عن تحويل البيانات الجافة (Raw Data) إلى تقارير بصرياً مذهلة باستخدام تنسيقات HTML و Progress Bars.

### 2️⃣ الوكلاء التلقائيون (Agents - `/agents`)
وكلاء متخصصون في مهام معينة:
- **`monitor_agent.py`**: وكيل المراقبة. يجمع بيانات استهلاك المعالج، الذاكرة، القرص، والشبكة لحظة بلحظة.
- **`docker_agent.py`**: وكيل Docker. متخصص في التفاعل مع Docker Engine لجلب حالة الحاويات وإدارتها (Start/Stop/Logs).
- **`deploy_agent.py`**: وكيل النشر. يدير عمليات Git (Status, Pull, Push) لتحديث المشروع تلقائياً من GitHub.

### 3️⃣ الخدمات والواجهة (Services & UI)
- **`services/gemini_service.py`**: الواجهة البرمجية للاتصال بمحركات جوجل للذكاء الاصطناعي.
- **`webapp/`**: يحتوي على "تطبيق تلجرام المصغر" (Mini App) الذي يوفر لوحة تحكم ويب بصرية متكاملة تعمل داخل التطبيق.
- **`api_server.py`**: خادم Flask بسيط يربط واجهة الويب ببيانات السيرفر الحقيقية.

### 4️⃣ ملفات التشغيل الأساسية
- **`main.py`**: الملف الرئيسي لتشغيل البوت بنظام واجهة المستخدم (UI) والأزرار.
- **`smart_bot.py`**: نسخة بديلة للتشغيل تركز على وضعية "الذكاء الصرف" (Smart Mode).
- **`telegram_agent.py`**: وكيل موحد يجمع كل المكونات في واجهة برمجية واحدة.

## 🛠️ كيف يعمل النظام؟ (Workflow)
1. **الاستلام**: يستقبل النظام أمرك (نصي أو عبر زر).
2. **التحليل**: يقرر `NexumInterpreter` ما إذا كان الأمر يحتاج إلى (تخطيط ذكي، تنفيذ مباشر، أو دردشة عادية).
3. **التخطيط**: يقوم `AIPlanner` ببناء خطوات التنفيذ إذا لزم الأمر.
4. **الفحص الأمني**: يمر كل أمر عبر `SecurityValidator` للتأكد من سلامته.
5. **التنفيذ**: يقوم `Executor` بتشغيل الأوامر وإرجاع النتائج.
6. **التنسيق**: يقوم `NexumFormatter` بتجميل النتائج وإرسالها لك.

1. قم بإنشاء بيئة افتراضية: `python -m venv venv`

4. ابدأ التشغيل: `python main.py`
