## ⚡ 2. الميزات والقدرات السيادية الجديدة (Core Advanced Capabilities)

لقد تمت ترقية النظام بالكامل في إصدار **v11.0.0** ليشمل القدرات الفريدة التالية:

### 1️⃣ محرك التطور والترميم الذاتي (🧬 Sovereign Self-Evolution Engine)
نظام تصحيح ذاتي مستمر يعمل في الخلفية لمراقبة سلامة الأنظمة وسد الثغرات التشغيلية:
* **الترميم الذاتي (Self-Healing Code Repair):** يقوم بمسح سجلات الأخطاء (`storage/logs/err.log`) لتتبع أي انهيارات أو استثناءات بايثون في الوكلاء المولدين. يقوم تلقائياً بقراءة كود الوكيل وتمريره مع الخطأ لـ Gemini، لإعطاء كود مصلح بالكامل (تجنب أخطاء الاستيراد والمكتبات) وحفظه تلقائياً.
* **تحليل فجوات القدرات (Capability Gap Scan):** يحلل سياق المحادثات الأخيرة للمدير في الخلفية للتعرف على أي مأموريات أو أدوات مطلوبة وغير مدعومة.
* **تخليق وتوليد الوكلاء (Agent Synthesis):** يقوم تلقائياً بإنشاء وكلاء بايثون مخصصين وجدد تحت مجلد `agents/generated/` ويسجلهم مباشرة لحل الفجوات المكتشفة على الفور.

### 2️⃣ سحابة جوجل ومنصة الوكلاء (☁️ Google Cloud Vertex AI integration)
تكامل حقيقي وعميق مع بيئة جوجل السحابية للمؤسسات والشركات:
* **استدعاء الدوال السحابية (Function Calling):** يقوم بربط الأدوات المحلية والترمنال ونظام الملفات السحابي بـ Vertex AI لتمكين Gemini من استدعاء وتنفيذ المهام التفاعلية مباشرة.
* **النشر السحابي (Cloud Run Docker Deployer):** خط أنابيب مستقل يقوم بتعبئة وكود أي وكيل يتم تخليقه داخل حاوية Docker مخصصة ونشره كخدمة ويب متناهية الصغر على منصة Google Cloud Run بنقرة زر واحدة.

### 3️⃣ محرك بروتوكولات العمل (📜 YAML Protocols Engine)
منظّم مسار مهام معقد وتدفقات عمل متتالية (DAG) مبني على لغة YAML:
* **التبعيات والشروط:** تنفيذ متسلسل للخطوات البرمجية، حيث تعتمد خطوة على مخرجات خطوة سابقة مع إمكانية وضع شروط تشغيل منطقية ديناميكية.
* **المزامنة والحل التلقائي للرموز:** استبدال تلقائي للمتغيرات بصيغة `{{goal}}` وربطها الفوري بـ `tool_registry` لتنفيذ الأدوات محلياً وسحابياً.

### 4️⃣ لوحة التحكم المصغرة المطورة (🌐 Unified Mini-App Dashboard)
ترقية شاملة لواجهة التحكم الخاصة بالتطبيق المصغر (WebApp) لتدعم سمات الـ Glassmorphism والتنقل المرن:
* **🧬 التطور الذاتي:** تشغيل فحص التطور اليدوي وعرض إحصائيات الوكلاء المصلحين والمخلقين لحظياً.
* **📜 البروتوكولات:** محرر YAML متكامل لكتابة وحفظ البروتوكولات، مع لوحة تشغيل تفاعلية تدعم إدخال سياق JSON لمراقبة نتائج التنفيذ خطوة بخطوة.
* **☁️ السحاب والـ API:** لوحة اختبار فوري لاستدعاء الدوال السحابية ومراقبة تفويض ADC وحالة المشروع، ونشر الوكلاء على Cloud Run.

## 🚀 4. التشغيل والبدء الفوري (Instant Standup)

### 1. تثبيت المتطلبات المعقمة:
```bash
pip install -r requirements.txt
```

### 2. إعداد ملف التكوين السري `.env`:
```env
TELEGRAM_TOKEN=your_bot_token_here
ADMIN_ID=your_telegram_numeric_id
GOOGLE_API_KEY=your_gemini_api_key_here

# تفعيل محرك سحابة جوجل (Vertex AI) اختيارياً
GOOGLE_GENAI_USE_VERTEXAI=true
GOOGLE_CLOUD_PROJECT=your_gcp_project_id
GOOGLE_CLOUD_LOCATION=us-central1
```

### 3. التشغيل الإنتاجي الموحد والخلفي (الوضع السيادي الفائق) عبر **PM2**:
يقوم ملف النظام بتشغيل كل من البوت النواة وخادم الـ API معاً لضمان مزامنة لوحة التحكم:

```bash
# تشغيل كل العمليات بالكامل (البوت + الـ REST API)
pm2 start ecosystem.config.js

# لمراقبة الحالة الحية
pm2 status

# لمشاهدة سجلات الحية للبوت
pm2 logs nexum-core

# لمشاهدة سجلات الـ API ولوحة التحكم
pm2 logs nexum-api
```
