## System Context (For External Agents)
هذا النظام هو **Nexum-Core**، وهو نظام تشغيل وكيل سيادي (Sovereign Agentic OS). يهدف النظام إلى الأتمتة الكاملة للعمليات المعقدة مع الحفاظ على السيادة التامة على البيانات والأمن.

### المبادئ التشغيلية الأساسية:
1. **YAMLProtocolEngine**: محرك لتنفيذ سير العمل عبر ملفات YAML لضمان قابلية التكرار والأمان.
2. **Self-Healing Engines**: محرك إصلاح ذاتي يعتمد على رصد الأخطاء وإعادة تهيئة العمليات تلقائياً.
3. **Sovereign Operations**: إدارة البنية التحتية وتأمينها عبر HMAC signing، دون الحاجة للتدخل البشري المستمر.
4. **Agent Registry**: نظام مركزي لتتبع العملاء (Agents) وأدوارهم.
5. **GCP Integration**: تكامل عميق مع Google Cloud Platform، خاصة Vertex AI للذكاء الاصطناعي التوليدي، مع استخدام بروتوكولات MCP (Model Context Protocol).

### دليل للمطورين والوكلاء (Developers & AI Agents Context):
- عند العمل على Nexum-Core، يجب استخدام **Absolute Paths** لجميع العمليات.
- يفضل دائماً استخدام **Self-healing protocols** قبل افتراض وجود فشل دائم.
- عند إضافة وكلاء (Agents)، يجب تسجيلهم في `storage/agent_registry.json`.
- الاتصال يتم عبر بروتوكولات مؤمنة. 
- يرجى الاطلاع على دليل البروتوكولات في `/docs/protocols/protocol_manual.md` قبل تنفيذ أي تعديلات جذرية.
